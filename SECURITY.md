# Security Policy

## Reporting a Vulnerability

Please report security issues privately and include:

- A clear description of the issue
- Impact assessment
- Reproduction steps or proof of concept
- Affected version/tag

If possible, also propose mitigation ideas.

## Response Expectations

- Initial triage: within 5 business days
- Status update: as soon as impact is confirmed
- Fix target: based on severity and release schedule

## Scope Notes

- Do not post unpatched vulnerabilities in public issues.
- Prefer private disclosure first to protect downstream users.
